# growi-plugin-table-extended

GROWI のページ本文中の Markdown テーブルを**ヘッダークリックでソート**し、**キーワードで行を絞り込む**ことができる Script プラグインです。フィルタ一致箇所は**ハイライト表示**されます。

## 機能

### ソート

- テーブルヘッダー（`th`）をクリックすると該当列で昇順 → 降順 → 元の順 にソートされる
- 数値列・日付列・文字列列をデータ内容から自動判定してソート
- 数値はカンマ区切り（例: `1,234`）に対応
- 日付は `YYYY-MM-DD` / `YYYY/MM/DD` 形式に対応

### フィルタ

- テーブル上部に検索ボックスを表示（左端のじょうごうアイコンが目印）
- キーワードを入力すると含む行のみ表示される
- 大文字・小文字を無視してマッチ（`Tokyo` / `TOKYO` / `tokyo` は同じ結果）
- スペース（半角・全角）区切りで複数キーワードを AND 絞り込み
- フィルタ適用中はテーブル下部に「M / N 件」を表示。クリアすると非表示に戻る
- 一致したテキスト箇所が黄色背景でハイライトされる（ライト/ダークモード両対応、印刷時は非表示）
- ソートと組み合わせ可能（フィルタ後のソートで非表示行は非表示のまま維持）

### コピー

- フィルタバー右端のコピーボタンをクリックすると表示中の行を **CSV** でクリップボードにコピー
- **Shift+クリック** で **Markdown テーブル**形式でコピー
- フィルタで絞り込んだ状態でコピーすると表示行のみが対象（ヘッダーは常に含む）
- ソート後の並び順がそのままコピー結果に反映される
- `,` `"` を含むセルは RFC 4180 に従い自動エスケープ、`|` を含むセルは `\|` にエスケープ
- 成功時はフォーマット別アイコン（CSV: グリッド / MD: M↓）と緑色フラッシュでフィードバック
- `navigator.clipboard` 非対応環境ではボタン自体が表示されない
- `data-no-filter` 属性付きテーブルではコピーボタンも非表示

### スティッキーヘッダー

- テーブルをスクロールしても `thead`（ヘッダー行）が画面上部に固定され、列名を常に確認できる
- 固定中は薄い下方向の影（`box-shadow`）でヘッダーが浮いているように見える
- GROWI のナビバー（`#grw-contextual-sub-nav` など）を自動検出し、固定位置（`top`）をナビバー高さに合わせて自動設定する
- ナビバーの高さが変化した場合も `ResizeObserver` で動的に追従する
- 自動検出で位置がずれる場合は CSS 変数 `--gpte-sticky-top` で手動上書きも可能

  ```css
  /* 手動で固定位置を上書きする例 */
  :root { --gpte-sticky-top: 55px; }
  ```

- `data-no-sticky` 属性を付けたテーブルはスティッキーヘッダーを無効化できる（ソート/フィルタは有効）

  ```html
  <table data-no-sticky>...</table>
  ```

### 視覚スタイル

- テーブル全体に視覚スタイルを付与（角丸・ヘッダー色・偶数行ストライプ・行ホバーハイライト）
- 色は GROWI のテーマ色（`--bs-primary`）を `color-mix()` で取り込み、テーマに追従する
- ダークモード対応：OS の `prefers-color-scheme: dark` と GROWI の UI トグル（`html[data-bs-theme="dark"]` / Bootstrap 5.3）の両方で配色が切り替わる

### その他

- SPA ページ遷移時に新ページのテーブルを自動検知・初期化する
- 閲覧モードのみ動作（編集モードでは無効）
- GROWI 標準のテーブル編集モーダル（`body.modal-open`）が開いている間はソート・フィルタをすべて停止し、編集 UI を妨げない
- 管理画面（`/admin` 以下）では無効
- 印刷時は矢印・フィルタ UI を非表示
- `data-no-sort` 属性を付けたテーブルはソート・フィルタ対象外にできる
- `data-no-filter` 属性を付けたテーブルはフィルタのみ無効化できる（ソートは有効）
- プラグイン無効化時にすべてのリスナー・DOM 変更・フィルタ UI を完全クリーンアップ

## インストール

GROWI の管理画面 `/admin/plugins` でこのリポジトリの URL を入力してインストールしてください。

インストール後、プラグインを有効化するとテーブルのヘッダーがクリック可能になります。

> **注意**: コード更新を反映する際は、管理画面でプラグインを**削除 → 再インストール**してください。有効/無効のトグルのみでは zip が取り直されません。

## 使い方

### 基本的なソートとフィルタ

Markdown でテーブルを書くだけで自動的にソート・フィルタ機能が有効になります。

```markdown
| 名前   | 年齢 | 入社日     |
|--------|------|------------|
| 田中   | 30   | 2020-04-01 |
| 鈴木   | 25   | 2022-08-15 |
| 佐藤   | 35   | 2018-01-10 |
```

- ヘッダーをクリック → 列ソート
- 上部の検索ボックスに「田中」と入力 → 「田中」行のみ表示
- 「田中 2020」と入力 → 両方を含む行のみ表示（AND 絞り込み）

### ソート・フィルタを無効にしたいテーブル

HTML の `data-no-sort` 属性をテーブルに付与するとソート・フィルタ両方の対象外にできます。フィルタのみ無効化したい場合は `data-no-filter` を使います（Markdown では直接指定不可のため、HTML ブロックとして記述してください）。

```html
<!-- ソートもフィルタも無効 -->
<table data-no-sort>
  ...
</table>

<!-- フィルタのみ無効（ソートは有効） -->
<table data-no-filter>
  ...
</table>

<!-- スティッキーヘッダーのみ無効（ソート/フィルタは有効） -->
<table data-no-sticky>
  ...
</table>
```

## 開発・ビルド

### セットアップ

```bash
pnpm install
pnpm approve-builds --all   # 初回のみ（pnpm 11+ の場合）
pnpm install                 # esbuild の postinstall を実行するため再度
```

### ビルド

```bash
pnpm build
```

`dist/manifest.json` と `dist/assets/` が生成されます。`dist/` はコミット必須です（GROWI はビルドを実行しないため）。
